# 카카오톡 대화 공유·임시보관함 개발 문서

## 1. 문서 정보

- 상태: 개발 예정
- 대상: Android 공유 수신 앱 + NEO Planner 서버/웹앱
- 작성일: 2026-07-18
- 기준 시간대: Asia/Seoul (`+09:00`)

## 2. 목표

카카오톡의 `대화 내용 내보내기`에서 생성된 `KakaoTalkChats.txt`를 Android 공유 메뉴를 통해 NEO Planner 앱으로 전달한다. 공유 직후에는 일정을 자동 생성하지 않고 대화 파일을 **임시보관함**에만 저장한다.

사용자는 나중에 NEO Planner의 임시보관함을 열어 일정 제목과 날짜·시간을 직접 선택한 후 최종적으로 일정을 등록한다.

핵심 원칙은 다음과 같다.

- 공유 직후 자동 일정 등록 금지
- 공유 파일의 안전한 임시 저장
- 사용자가 날짜·시간을 확인한 뒤 명시적으로 등록
- 동일 파일 중복 저장 방지
- 원본 대화와 생성된 일정의 연결 관계 보존

## 3. 사용자 흐름

```text
카카오톡 대화 내용 내보내기
        ↓
Android 공유 목록에서 NEO Planner 선택
        ↓
KakaoTalkChats.txt 읽기 및 서버 전송
        ↓
임시보관함에 pending 상태로 저장
        ↓
사용자가 나중에 임시보관함 열기
        ↓
일정 제목·날짜·시작/종료 시간 입력
        ↓
일정 등록
        ↓
임시보관 항목을 registered 또는 archived로 변경
```

### 3.1 공유 직후 화면

- 파일명 표시
- 대화방 이름을 파일 내용에서 추출해 표시
- 파일 크기 및 공유 시각 표시
- `임시보관함에 저장 중` 진행 상태 표시
- 저장 성공 후 `임시보관함에 저장했습니다` 안내
- 일정 분석이나 등록 화면을 강제로 열지 않음

### 3.2 임시보관함 목록

각 항목에 다음 정보를 표시한다.

- 대화방 이름
- 파일명
- 저장 시각
- 대화 기간
- 파일 크기 또는 문자 수
- 상태: `대기`, `일정 등록 완료`, `보관`
- 생성된 일정 개수

정렬 기본값은 최근 저장 순이다.

### 3.3 임시보관함 상세

- 원본 대화 검색 및 스크롤
- 일정으로 사용할 대화 구간 선택
- 일정 제목 입력
- 날짜 선택
- 시작 시간 선택
- 종료 시간 선택
- 카테고리와 우선순위 선택
- 메모 입력
- `일정 등록` 버튼

한 개의 대화 파일에서 여러 일정을 반복해서 등록할 수 있어야 한다.

## 4. MVP 범위

### 포함

- Android 공유 대상 앱 등록
- `.txt` 및 `text/plain` 수신

- 임시보관함 목록/상세 조회
- 사용자가 날짜·시간을 직접 입력해 일정 등록
- 생성된 일정과 임시보관 항목 연결
- 중복 공유 방지
- 임시보관 항목 삭제 및 보관 처리

### 제외

- 카카오톡 알림 실시간 감지
- 공유 즉시 LLM 자동 분석
- 사용자 확인 없는 자동 일정 등록
- 이미지·동영상·음성 파일 분석

- 외부 공개 네트워크에서의 무인증 사용

LLM 일정 추천 기능은 후속 단계에서 임시보관함 상세 화면의 선택 기능으로 추가한다.

## 5. Android 앱 설계

### 5.1 공유 인텐트 등록

Android 앱의 `AndroidManifest.xml`에 `ACTION_SEND`와 `text/plain` 인텐트 필터를 등록한다.

```xml
<activity
    android:name=".ShareReceiverActivity"
    android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.SEND" />
        <category android:name="android.intent.category.DEFAULT" />
        <data android:mimeType="text/plain" />
    </intent-filter>
</activity>
```

앱 이름과 아이콘은 공유 목록에서 목적이 분명하도록 다음과 같이 사용한다.

- 앱 이름: `NEO Planner`
- 공유 화면 라벨: `플래너 임시보관함에 저장`

### 5.2 공유 데이터 수신

카카오톡이 파일을 `Intent.EXTRA_STREAM`의 `content://` URI로 전달하는 경우를 우선 처리한다. 기기나 카카오톡 버전에 따라 `Intent.EXTRA_TEXT`가 포함될 가능성도 고려한다.

처리 순서는 다음과 같다.

1. `Intent.ACTION_SEND` 확인
2. MIME 타입이 `text/plain` 또는 `text/*`인지 확인
3. `EXTRA_STREAM` URI 확인
4. `ContentResolver.openInputStream()`으로 즉시 파일 읽기
5. UTF-8 우선, 실패 시 한국어 텍스트 인코딩 감지
6. 파일 크기와 텍스트 길이 제한 확인
7. 서버 임시보관 API 호출
8. 성공/실패 결과 표시

공유 URI의 읽기 권한은 일시적일 수 있으므로 앱이 실행된 직후 내용을 읽거나 앱 캐시에 복사해야 한다.

### 5.3 Android 로컬 상태

네트워크 실패에 대비해 전송 대기 큐를 둔다.

```text
RECEIVED → UPLOADING → SAVED
                    ↘ FAILED → RETRYING
```

- 네트워크가 없으면 로컬 캐시에 암호화하여 임시 저장
- 재시도는 WorkManager 사용
- 서버 저장 성공 후 로컬 원문은 삭제 가능
- 사용자가 앱 설정에서 로컬 보관 기간을 선택할 수 있도록 확장 가능

## 6. 서버 데이터 모델

현재 `data/db.json`에 `todos`와 별도로 `chatImports` 컬렉션을 추가한다.

```json
{
  "todos": [],
  "chatImports": [
    {
      "id": "import_id",
      "source": "kakaotalk-export",
      "fileName": "KakaoTalkChats.txt",
      "chatRoomName": "현명♥ 님과 카카오톡 대화",
      "rawText": "원본 대화 내용",
      "contentHash": "sha256",
      "characterCount": 351527,
      "conversationStartedAt": "2026-01-16T15:29:00+09:00",
      "conversationEndedAt": "2026-07-16T17:13:00+09:00",
      "sharedAt": "2026-07-18T15:21:53+09:00",
      "status": "pending",
      "createdTodoIds": [],
      "createdAt": "2026-07-18T15:21:53+09:00",
      "updatedAt": "2026-07-18T15:21:53+09:00"
    }
  ]
}
```

### 6.1 상태 값

| 상태 | 의미 |
|---|---|
| `pending` | 저장됐지만 아직 일정이 등록되지 않음 |
| `in_progress` | 사용자가 상세 화면에서 작업 중 |
| `registered` | 하나 이상의 일정이 등록됨 |
| `archived` | 작업 완료 후 보관됨 |

`registered` 상태에서도 같은 대화에서 추가 일정을 만들 수 있다.

## 7. API 설계

### 7.1 임시보관 항목 생성

`POST /api/chat-imports`

```json
{
  "source": "kakaotalk-export",
  "fileName": "KakaoTalkChats.txt",
  "rawText": "카카오톡 대화 내용",
  "sharedAt": "2026-07-18T15:21:53+09:00"
}
```

응답:

```json
{
  "id": "import_id",
  "status": "pending",
  "duplicate": false,
  "chatRoomName": "현명♥ 님과 카카오톡 대화",
  "characterCount": 351527,
  "createdAt": "2026-07-18T15:21:53+09:00"
}
```

서버는 `rawText`의 SHA-256 해시를 계산한다. 동일 해시가 이미 존재하면 새 항목을 만들지 않고 기존 항목과 `duplicate: true`를 반환한다.

### 7.2 임시보관 목록

`GET /api/chat-imports?status=pending`

- 기본 정렬: `createdAt DESC`
- 목록 응답에는 `rawText`를 포함하지 않음
- 추후 페이지네이션 적용

### 7.3 임시보관 상세

`GET /api/chat-imports/:id`

- 원본 대화 포함
- 연결된 일정 목록 포함

### 7.4 임시보관 대화에서 일정 등록

`POST /api/chat-imports/:id/todos`

```json
{
  "title": "병원 예약",
  "content": "카카오톡 대화에서 수동 등록",
  "startDate": "2026-07-20T14:00:00+09:00",
  "endDate": "2026-07-20T15:00:00+09:00",
  "category": "personal",
  "priority": "medium",
  "color": "#06b6d4",
  "sourceExcerpt": "월요일 오후 2시에 병원 예약했어"
}
```

서버 처리:

1. 임시보관 항목 존재 여부 확인
2. 필수 일정 필드 및 날짜 검증
3. `todos`에 일정 생성
4. `createdTodoIds`에 ID 추가
5. 임시보관 상태를 `registered`로 변경
6. 생성된 일정 반환

### 7.5 상태 변경

`PATCH /api/chat-imports/:id`

```json
{
  "status": "archived"
}
```

### 7.6 삭제

`DELETE /api/chat-imports/:id`

기본 동작은 임시보관 원본만 삭제하고 이미 생성된 일정은 유지한다. 연결 일정까지 함께 삭제하는 기능은 별도 확인 UI가 있을 때만 제공한다.

## 8. 웹앱 화면 설계

### 8.1 내비게이션

사이드바에 `카톡 임시보관함` 메뉴를 추가한다.

표시 배지:

- `pending` 항목 개수
- 신규 공유 항목이 생기면 강조 표시

### 8.2 임시보관 목록 화면

- 카드 또는 테이블 형태
- 대화방 이름, 저장일, 대화 기간, 상태 표시
- 검색 및 상태 필터
- 상세 열기, 보관, 삭제 동작

### 8.3 일정 등록 화면

- 왼쪽: 원본 대화
- 오른쪽: 일정 입력 폼
- 모바일에서는 원본 대화와 입력 폼을 탭으로 분리
- 선택한 대화 문장을 `sourceExcerpt`로 자동 입력
- 시작일보다 종료일이 빠르면 저장 차단
- 저장 성공 후 원본 대화 화면을 유지해 연속 등록 지원

## 9. 날짜·시간 규칙

- 앱과 서버는 ISO 8601 및 `+09:00` 오프셋 사용
- Android 날짜/시간 선택기는 기기의 로컬 시간으로 표시
- 날짜 미선택 상태에서는 일정 등록 버튼 비활성화
- 종료 시간 기본값은 시작 시간 + 1시간
- 종료 시간이 시작 시간 이하이면 오류 표시
- 사용자가 직접 고른 값은 LLM이나 서버가 다시 변경하지 않음

## 10. 보안 및 개인정보

카카오톡 원문에는 민감한 개인정보가 포함될 수 있으므로 다음 조건을 적용한다.

- 개발 중에도 API 인증 토큰 사용
- 운영 환경에서는 HTTPS 사용
- 현재 `http://192.168.219.112:19000`은 같은 내부망의 개발 환경에서만 허용
- 서버 로그에 `rawText` 출력 금지
- 오류 로그에 대화 전문 포함 금지
- 임시보관 삭제 시 원문을 실제 데이터에서 제거
- 자동 보관 기간 설정을 후속 기능으로 제공
- Android 캐시에 남은 원문은 서버 저장 성공 후 삭제

## 11. 오류 처리

| 상황 | 동작 |
|---|---|
| 지원하지 않는 파일 | `.txt` 파일만 지원한다는 안내 |
| 빈 파일 | 저장하지 않고 오류 표시 |
| 허용 크기 초과 | 파일 크기 제한 안내 |
| 네트워크 연결 실패 | 로컬 전송 대기 큐에 저장 후 재시도 |
| 서버 인증 실패 | 로그인 또는 연결 설정 안내 |
| 동일 파일 재공유 | 기존 임시보관 항목 열기 제공 |
| 손상된 인코딩 | 인코딩 재시도 후 실패 시 사용자 안내 |
| 일정 저장 실패 | 입력값 유지 후 재시도 제공 |

## 13. 완료 조건

- 카카오톡 내보내기 공유 목록에 `NEO Planner`가 표시된다.
- `KakaoTalkChats.txt`를 선택하면 임시보관함에 한 번만 저장된다.
- 공유 직후 일정이 자동 생성되지 않는다.
- 사용자는 나중에 임시보관함에서 날짜와 시간을 직접 선택할 수 있다.
- 한 대화 파일에서 여러 일정을 등록할 수 있다.
- 등록한 일정은 기존 달력과 목록 화면에 정상 표시된다.
- 취소하거나 실패해도 사용자가 입력한 일정 정보가 사라지지 않는다.
- 원본 대화를 삭제해도 이미 만든 일정은 유지된다.
- 서버와 Android 캐시에서 원본 대화를 삭제할 수 있다.

## 14. 후속 기능

- 공유 완료 푸시 알림
- 임시보관 원문 자동 삭제 기간
- 서버 외부 접속용 HTTPS 또는 VPN 구성
- Android 홈 화면 위젯 및 공유 바로가기
