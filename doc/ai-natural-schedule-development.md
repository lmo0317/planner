# AI 자연어 일정 추가 개발 문서

## 1. 기능 개요

사용자가 원하는 일정을 자연어로 입력하면 LLM이 제목, 날짜, 시작·종료 시간, 카테고리, 우선순위를 구조화한다. 분석 결과는 즉시 저장하지 않고 선택 가능한 미리보기 목록으로 보여준다. 사용자는 제목과 시간을 확인·수정한 뒤 선택한 일정만 등록한다.

예시:

```text
다음 주 화요일 오후 3시에 강남 치과 예약하고,
수요일 오전 10시에는 팀 회의 추가해줘
```

## 2. 사용자 흐름

```text
사이드바의 AI 일정 추가 선택
→ 자연어 일정 입력
→ 일정 확인
→ AI 분석 결과 목록 표시
→ 제목·시작·종료 시간 검토 및 수정
→ 등록할 일정 선택
→ 선택 일정 등록
→ 달력과 목록 새로고침
```

## 3. UI 구성

- 사이드바 메뉴: `AI 일정 추가`
- 입력 모달
  - 자연어 입력창
  - 예시 문장 버튼
  - `일정 확인` 버튼
- 분석 결과
  - 일정별 선택 체크박스
  - 수정 가능한 제목
  - 수정 가능한 시작·종료 일시
  - 날짜 판단 이유
  - 카테고리, 우선순위, 신뢰도
  - `선택 일정 등록` 버튼
- 날짜가 빠진 경우 확인 질문 표시
- 시간이 빠진 경우 종일 일정으로 표시

## 4. API

### 자연어 일정 분석

`POST /api/todos/parse-natural-language`

요청:

```json
{
  "text": "내일 오후 3시에 강남 치과 예약 추가해줘",
  "baseDate": "2026-07-18T10:00:00+09:00"
}
```

응답:

```json
{
  "events": [
    {
      "title": "강남 치과 예약",
      "content": "강남 치과 예약",
      "startDate": "2026-07-19T15:00:00+09:00",
      "endDate": "2026-07-19T16:00:00+09:00",
      "priority": "medium",
      "category": "personal",
      "color": "#06b6d4",
      "dateReason": "내일을 기준일 다음 날로 계산하고 오후 3시를 적용",
      "confidence": 1
    }
  ],
  "clarification": ""
}
```

시간 누락 예시:

```json
{
  "events": [
    {
      "title": "프로젝트 회의",
      "startDate": "2026-07-20T00:00:00+09:00",
      "endDate": "2026-07-20T23:59:59+09:00",
      "allDay": true
    }
  ],
  "clarification": ""
}
```

## 5. 날짜·시간 처리 규칙

- 기준 시각은 브라우저의 로컬 시간과 오프셋을 포함해 전달한다.
- `오늘`, `내일`, `모레`, `이번 주`, `다음 주`는 서버가 `DATE_HINT`로 선계산한다.
- LLM은 서버가 계산한 `DATE_HINT`를 변경할 수 없다.
- 날짜가 있어야 일정 후보를 생성한다.
- 시간이 없으면 임의의 시간을 만들지 않고 `allDay: true`인 종일 일정으로 생성한다.
- 종일 일정은 해당 날짜의 `00:00:00`부터 `23:59:59`까지로 정규화한다.
- `월요일부터 금요일까지` 같은 기간 표현은 날짜별로 나누지 않고 시작일과 종료일을 가진 일정 한 건으로 생성한다.
- 월간 달력에서는 기간 일정이 같은 주의 동일한 레인에서 연결된 막대로 표시된다.
- 종료 시간이 없으면 시작 시간에서 1시간 뒤로 설정한다.
- 결과는 ISO 8601과 `+09:00` 오프셋으로 정규화한다.
- 사용자가 미리보기에서 수정한 값은 다시 AI가 변경하지 않는다.

## 6. 검증 규칙

- 빈 입력 거부
- 최대 입력 길이 4,000자
- 날짜 표현 누락 시 확인 질문
- 시간 표현 누락 시 종일 일정 생성
- 시작 시간이 종료 시간 이상이면 등록 차단
- 제목이 비어 있으면 등록 차단
- 신뢰도 0.65 미만 일정 제거
- 동일 제목·시작 시간 일정 중복 제거
- JSON Schema로 LLM 응답 형식 강제

## 7. 테스트

테스트 파일: `test_natural_parse.py`

회귀 항목:

- `내일` 단일 일정
- `다음 주` 복수 일정
- 명시적인 월·일 일정
- 시간 누락 시 종일 일정 판정
- 분석 → 일정 생성 → 조회 → 삭제 통합 흐름

실행:

```bash
python test_natural_parse.py http://127.0.0.1:19000/api/todos/parse-natural-language
```

## 8. 완료 조건

- 사이드바에서 AI 일정 추가 모달을 열 수 있다.
- 자연어 한 문장에서 여러 일정을 추출할 수 있다.
- 분석 결과가 저장 전 미리보기로 표시된다.
- 사용자가 일정별 선택 여부를 변경할 수 있다.
- 제목과 시작·종료 시간을 수정할 수 있다.
- 날짜가 없으면 임의 등록하지 않고, 시간만 없으면 종일 일정으로 등록한다.
- 선택된 일정만 기존 `todos` API로 저장된다.
- 저장 후 달력과 목록에 즉시 반영된다.
